# 토스 자동매매 Go-Live 기준

작성: 2026-06-23

## 전제 (반드시 기억할 것)

백테스트 절대수익(22% CAGR)은 **NDX100 생존편향으로 부풀려진 거품**이다. 실제 수익
엔진인 전략 B(모멘텀 단독 20%)가 "오늘의 나스닥 승자를 과거부터 산" 효과에 노출돼
있다. 검증된 사실은 두 가지뿐:

1. **위험관리는 진짜** — 5개 시대 전부 바이앤홀드 대비 Sharpe 우위 + 낮은 낙폭
2. **파라미터는 과최적화 아님** — 워크포워드에서 IS 최적(RSI35, B_ATR4)이 OOS도 1위

→ **기대수익은 백테스트의 절반 이하로 깔고 시작한다.** 실제 검증은 forward 뿐.

## 현재 단계

- ✅ Phase 1 드라이런 실행기(`toss_execution.py`) — 실주문 0
- ✅ 페이퍼 forward 검증(`forward_test.py`) — 토스 실시간 시세로 가상 운용, 일별 누적
- ⬜ Phase 2 실주문 — 아래 Go-Live 기준 전부 통과 전엔 금지

## Go-Live 기준 (실제 돈 전환 — 전부 AND 조건)

매일 `python scanner_v4.py && python forward_test.py`를 돌려 forward 곡선을 쌓고,
아래를 **모두** 만족할 때만 소액 실거래로 전환한다.

| # | 기준 | 임계값 |
|---|---|---|
| 1 | **forward 운용 기간** | 최소 **8주**(평일 기준 ~40 거래일) 연속 기록 |
| 2 | **forward 수익** | 누적 평가액이 시작 자본 대비 **≥ −5%** (큰 손실 없이 생존) |
| 3 | **forward 낙폭(MDD)** | forward 기간 최대낙폭 **≤ 15%** |
| 4 | **신호 sanity** | 드라이런 의도 주문에 비상식적 종목/수량/가격 **0건** (매주 눈으로 점검) |
| 5 | **체결 현실성** | 페이퍼 체결가 vs 실제 호가 괴리가 합리적 (슬리피지 감당 가능) |
| 6 | **자금 성격** | **잃어도 생활에 지장 없는 돈** 인가 (가장 중요) |

하나라도 미달이면 forward 연장 또는 중단. "거의 됐다"는 통과 아님.

## Phase 2 전환 시 (기준 통과 후)

1. **`TossClient.create_order()` 구현** — `POST /api/v1/orders` 스키마 확인 후
2. **`toss_execution.py --live` 활성화** — 단, 첫 주문은 **수동 1회** 소액 검증
3. **시작 비중: 전체 가용자금의 5%** (한 번에 풀베팅 금지)
4. **킬스위치 실테스트** — `touch kill_switch.flag`로 실제 중단 확인
5. **일일 손실 한도** — 전일 평가액 기준선 영속화 후 −3% 작동 확인

## 운영 중 서킷 브레이커 (실거래 시)

- forward/실거래 누적 −15% 도달 → 자동 중단(킬스위치) 후 재검토
- 단일일 −3% → 당일 신규진입 차단 (이미 구현)
- 월 1회 forward vs 실거래 성과 대조 — 괴리 크면 점검

## 정공법 (수익을 진짜 높이려면)

파라미터 튜닝은 막다른 길(16개 중 1개만 통과). 실질 개선은:
- **시점별 구성종목**으로 B의 진짜 엣지 확인 (외부 데이터 필요)
- 기존과 **상관 낮은 새 알파원** 추가
- C(VIX패닉)는 단독 무의미 → A/B 위 오버레이로만
